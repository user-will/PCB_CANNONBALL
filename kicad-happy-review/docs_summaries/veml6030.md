# VEML6030 — Application Summary

**Manufacturer:** Vishay Semiconductors
**Type:** High-accuracy 16-bit ambient light sensor (ALS) with I²C interface and programmable interrupt
**Package:** 2.0 mm × 2.0 mm × 0.87 mm surface-mount, transparent (optical) — 6-pin
**Datasheet:** VEML6030, Vishay Semiconductors (`docs/components/veml6030/veml6030.md`)
**Related application note:** "Designing VEML6030 into an Application" (Vishay AN — referenced by datasheet for interrupt setup and resolution tables)

> This summary is a reference for validating a board design that uses the **VEML6030**. Packaging, tape-and-reel, soldering/reflow, mechanical-dimension, and register/firmware detail are omitted except where they impose an electrical or layout requirement on the board. Where a measurement mode forces a specific electrical or addressing configuration, that configuration is noted.

## 1. Short Description

The VEML6030 is a **digital ambient light sensor** integrating a high-sensitivity photodiode, a low-noise amplifier, a 16-bit A/D converter, and an I²C interface in a miniature transparent 2 × 2 mm package. It outputs a calibrated 16-bit ambient-light value (plus a separate WHITE channel) over I²C — no analog front-end or external ADC is needed on the host.

Defining features for board design:

- **Single supply** V_DD = 2.5 V … 3.6 V (3.3 V typ).
- **Separate I²C bus-voltage domain:** the I²C H-level range is **1.7 V … 3.6 V**, so SDA/SCL pull-ups may be referenced to a logic rail different from V_DD (e.g. 1.8 V host).
- **Very low power:** ~0.5 µA standby; 45 µA typical in continuous operation (PSM disabled); down to single-digit µA in power-saving modes.
- **Wide dynamic range:** 0 lx to ~140 klx, resolution down to 0.0042 lx/count.
- **Open-drain interrupt (INT) pin** with programmable high/low threshold windows — requires a pull-up.
- **Selectable I²C address** via the ADDR pin (0x48 when high / 0x10 when low, 7-bit), allowing two devices on one bus.
- **100 Hz / 120 Hz flicker rejection** and on-chip temperature compensation.

**Typical applications:** display/backlight dimming in mobile devices, industrial on/off lighting control, and optical switching for consumer, computing, and industrial displays.

### 2.4 Optical Characteristics (affect placement / aperture)

- **Spectral response** closely matches the human eye (peak in the visible band; see datasheet Fig. 5–6); a separate WHITE channel is available for light-source compensation.
- **Angular field of view:** relative sensitivity ≈ 1.0 on-axis, falling to ≈ 0.5 at roughly **±50°…±55°** off-axis (datasheet Fig. 7). The optical window must have a clear, unobstructed cone over this angle.

## 3. Sample Applications & Reference Circuits

### 3.1 Pin Assignment (from Application Circuit, Fig. 8)

| Pin | Name | Function |
|---|---|---|
| 1 | GND | Ground |
| 2 | SDA | I²C serial data (open-drain) |
| 3 | INT | Interrupt output — **open-drain**, requires pull-up |
| 4 | ADDR | I²C address select: high (V_DD) → 0x48; low (GND) → 0x10 |
| 5 | SCL | I²C serial clock |
| 6 | V_DD | Supply voltage |

### 3.2 Typical Application Circuit (Datasheet Fig. 8)

| Component | Connection | Value | Purpose |
|---|---|---|---|
| C2 | V_DD → GND | **≥ 100 nF** | Primary decoupling — place close to V_DD (pin 6). Normally the only decoupling needed. |
| C1 | V_DD → GND | 10 µF | **Optional** bulk cap — only for "very disturbed supply" |
| R4 | supply → V_DD | 10 Ω | **Optional** series resistor — only for "very disturbed supply" (forms an RC filter with C1/C2) |
| R1 | SDA → bus rail (1.7–3.6 V) | 2.2 kΩ … 4.7 kΩ | I²C data pull-up |
| R2 | SCL → bus rail (1.7–3.6 V) | 2.2 kΩ … 4.7 kΩ | I²C clock pull-up |
| R3 | INT → bus rail | 10 kΩ … 100 kΩ | Interrupt pull-up (open-drain output) |

Connections:
- **V_DD** (pin 6) to the 2.5–3.6 V supply, **100 nF decoupling adjacent**; add R4 (10 Ω) + C1 (10 µF) only if the supply is noisy.
- **GND** (pin 1) to ground.
- **SDA / SCL** (pins 2 / 5) to the host I²C master, each pulled to the 1.7–3.6 V bus rail.
- **INT** (pin 3) to a host GPIO, pulled up (10 k–100 kΩ); leave the pull-up in place even if the interrupt is unused, or tie off appropriately.
- **ADDR** (pin 4) tied **high or low** to select the slave address (see §3.3).

> All pull-up resistors must be **> 1 kΩ**. SDA/SCL: 2.2–4.7 kΩ; INT: 10–100 kΩ.

### 3.3 Address Configuration (forced by ADDR pin)

| ADDR pin | 7-bit address | Write byte | Read byte |
|---|---|---|---|
| High (V_DD) | 0x48 | 0x90 | 0x91 |
| Low (GND) | 0x10 | 0x20 | 0x21 |

- The ADDR pin must be **statically tied** to V_DD or GND — not left floating.
- For **two VEML6030 on one bus**, strap one ADDR high and one low.
- The ID register (0x07) low byte = 0x81; high byte reflects the address option (0xD4 for 0x90/0x48 option, 0xC4 for 0x20/0x10 option) — useful for power-on verification but no board impact.

### 3.4 Configuration Notes With Electrical Relevance

- The device **powers up in shutdown** (ALS_SD = 1 default); firmware must clear it. No board impact, but explains zero current at power-on.
- **Power-saving modes (PSM)** trade refresh time for current (8 µA → 2 µA across modes 1–4 at IT = 100 ms). Choosing low current extends the measurement refresh interval — relevant only to power budgeting, not wiring.
- Finest resolution (0.0042 lx/step) requires **ALS_GAIN ×2, IT = 800 ms**; max range (140 klx) requires **ALS_GAIN ×1/8, IT = 25 ms**. These are register settings, not hardware changes.

## 4. Component Placement, Routing & Layout Best Practices

### 4.1 Optical Aperture & Placement (defining constraint for this part)

- The package top is the **optical window** — it must face the light to be measured with a **clear, unobstructed optical path**. Keep the aperture clean and free of solder mask, silkscreen, flux residue, and conformal coating over the window.
- Respect the **±50–55° field of view** (§2.4): no tall neighboring components, connectors, or enclosure walls inside the sensing cone, or they will shade/vignette the reading.
- If the sensor looks through a **cover lens / light pipe / tinted window**, account for its transmittance — the wide 16-bit range and "low-transmittance (dark) lens" support are intended for exactly this; calibrate gain/integration time to the window's attenuation.
- Keep the sensor away from **on-board light sources** (LEDs, backlights) whose stray/reflected light could couple into the window and corrupt ambient readings; add optical baffling if co-located.

### 4.2 Power & Decoupling

- Place the **100 nF decoupling cap (C2) directly at V_DD (pin 6)** with a short return to GND.
- Add the **optional R4 (10 Ω) + C1 (10 µF) RC filter** only for noisy/disturbed supplies; on a clean rail, 100 nF alone is sufficient.

### 4.3 I²C / Interrupt Routing

- Keep **SDA/SCL short** and choose pull-ups (2.2–4.7 kΩ) so the bus rise time meets the I²C-mode limit (≤ 300 ns Fast mode) for the actual bus capacitance.
- Pull SDA/SCL up to a rail within the **1.7–3.6 V** bus-voltage window (may differ from V_DD).
- The **INT pin is open-drain** — it needs its own pull-up (10–100 kΩ); route it to a host GPIO and keep it from coupling into the optical/analog area.
- Ensure a **solid common ground** between sensor and I²C master.

### 4.4 Thermal / Environmental

- Operating range is limited to **−25 °C … +85 °C** — keep the sensor away from hot components and verify the local PCB temperature stays in range.
- Standard ESD-sensitive handling applies (transparent optical SMD part).

## 5. Design-Validation Checklist (VEML6030)

- [ ] V_DD within **2.5–3.6 V** across all conditions; absolute max 4 V never exceeded.
- [ ] **100 nF** decoupling cap at V_DD (pin 6), short return; optional 10 Ω + 10 µF added only if the supply is noisy.
- [ ] SDA and SCL pull-ups present (**2.2–4.7 kΩ**, > 1 kΩ) to a **1.7–3.6 V** rail; bus rise time meets the I²C mode in use.
- [ ] **INT** pin has its own pull-up (**10–100 kΩ**) and routes to a host GPIO (open-drain).
- [ ] **ADDR** pin statically tied high or low for the intended slave address (0x48 / 0x10); no float; two-device buses use opposite straps.
- [ ] Optical window has a **clear, unobstructed cone (±~55°)** — no solder mask/silk/coating over it, no shading neighbors or enclosure walls.
- [ ] If a cover lens/window is used, its **transmittance** is accounted for in gain/integration-time selection.
- [ ] Sensor isolated from **stray on-board light** (LEDs/backlight) and from heat sources; local temperature within **−25…+85 °C**.
- [ ] Host firmware clears the power-on **shutdown** bit and selects gain/integration time appropriate to the lighting range.

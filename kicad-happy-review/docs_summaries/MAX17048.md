# MAX17048 — Application Summary

**Manufacturer:** Maxim Integrated (now Analog Devices)
**Type:** Micropower 1-Cell Li+ Fuel Gauge (battery state-of-charge monitor) with ModelGauge™ algorithm and I²C interface
**Package:** 8-pin TDFN-EP (2 mm × 2 mm) or 8-bump WLP (0.9 mm × 1.7 mm)
**Datasheet:** MAX17048/MAX17049, Maxim Integrated, Rev. 7 (11/16) (`docs/components/MAX17048/MAX17048-MAX17049.md`)

> This summary is a reference for validating a board design that uses the **MAX17048** (single-cell variant). The datasheet is shared with the **MAX17049** (2-cell variant); values below are filtered to the **MAX17048** wherever the two differ. Packaging, mechanical, soldering, and marking information is intentionally omitted. Register/software detail is included only where it imposes an electrical/connection requirement on the board.

---

## 1. Short Description

The MAX17048 is a tiny, ultra-low-power **current-less fuel gauge** for a single lithium-ion / Li-polymer cell. It continuously reports **state-of-charge (SOC)**, **cell voltage (VCELL)**, and **charge/discharge rate (CRATE)** to a host over I²C.

Its defining feature is the **ModelGauge algorithm**, which estimates SOC from cell **voltage alone** by simulating the battery's nonlinear dynamics. This means it:

- **Needs no current-sense resistor** in the battery path (unlike coulomb-counter gauges) — saving cost, board area, and series resistance;
- **Does not accumulate error** over time and **needs no learn cycles** or full charge/discharge correction events;
- **Auto-debounces battery insertion** (best of 16 samples) for a good initial SOC estimate;
- **Auto-enters/exits a 3 µA hibernate mode** at low load, fuel-gauging the whole time.

Because it is a voltage-sense device with **only one required external component** (a bypass capacitor), it can be located on either the **pack side** or the **system/host side**. Temperature compensation is handled by the host writing the `RCOMP` parameter — there is no on-chip temperature sensor.

**Typical applications:** smartphones, tablets, smartwatches/wearables, Bluetooth headsets, health/fitness monitors, cameras, medical devices, handheld terminals, wireless speakers, building-automation sensors.

### 2.4 Operating Modes & Power States

| Mode | Current (typ) | Entry | Notes |
|---|---|---|---|
| **Active** | 23 µA | Normal operation / high charge-discharge rate | ADC + SOC update every 250 ms |
| **Hibernate** | 3 µA (4 µA max) | Auto when \|CRATE\| < HibThr for >6 min, or forced | ADC + SOC update every 45 s; still fuel-gauging. Default = auto enter/exit |
| **Sleep** | < 1 µA | `MODE.EnSleep=1` then `CONFIG.SLEEP=1`, or bus held low > t_SLEEP | Halts all operation; **does not detect self-discharge** → SOC drifts. Wake before charge/discharge |

> For applications that can tolerate ~4 µA, the datasheet recommends **hibernate over sleep**, because hibernate keeps fuel-gauging and sleep does not.

### 3.2 Reference Application — 1S Cell Pack (Datasheet Fig. 14)

The MAX17048 reference circuit uses **exactly one external passive** — a single bypass capacitor:

| Component | Connection | Value | Purpose |
|---|---|---|---|
| C_BYP | `VDD`/`CELL` node → GND | **0.1 µF** | Supply bypass / sense-node filtering (only required external component) |
| R_SDA | `SDA` → bus rail | 2.2 k–10 kΩ (typ) | I²C pull-up (**system must provide**) |
| R_SCL | `SCL` → bus rail | 2.2 k–10 kΩ (typ) | I²C pull-up (**system must provide**) |
| R_ALRT | `ALRT` → rail | 10 kΩ (typ) | Open-drain alert pull-up, only if `ALRT` used |

Connections (from Fig. 14):

- **`VDD` and `CELL` are tied together** to the battery positive terminal (through the pack protection circuit). On the MAX17048, `VDD` is the actual sense node; `CELL` is not internally connected but is wired to the same node by convention.
- **`CTG` and `GND`** connect to the battery negative terminal / system ground.
- **`QSTRT` → GND** (quick-start unused — recommended for most designs; the IC handles startup transparently).
- **`ALRT` → host interrupt** input so the gauge can signal low-SOC / over-/under-voltage / reset / 1 % SOC-change events.
- **`SDA`/`SCL` → host I²C master**, with external pull-ups to the bus rail.

### 3.3 Application Configurations (Datasheet Table 3)

| System configuration | `VDD` | `ALRT` | `QSTRT` |
|---|---|---|---|
| 1S pack-side location | Power directly from battery | Leave unconnected | Connect to GND |
| 1S host-side location | Power directly from battery | Leave unconnected | Connect to GND |
| 1S host-side, low-cell interrupt | Power directly from battery | Connect to system interrupt | Connect to GND |
| 1S host-side, hardware quick-start | Power directly from battery | Leave unconnected | Connect to rising-edge reset signal |

> In **all** configurations the system must provide pull-ups for `SDA`, `SCL`, and (if used) `ALRT`.

### 3.4 Connection / Configuration Requirements Driven by Function

These are software/register items that **dictate an electrical or wiring choice** on the board:

- **Fixed I²C slave address `0x6C` (write) / `0x6D` (read)** — only one MAX17048 per I²C bus segment; for multiple gauges use a bus mux or separate buses.
- **Bus-low timeout:** if `SCL` and `SDA` are both held low > 2.5 s the IC enters sleep/shutdown. Do not park the bus low; ensure pull-ups are present so the idle bus is high.
- **VRESET (battery-swap detect):** for a **captive (non-removable) battery**, set VRESET to its minimum (≈2.5 V). For a **removable battery**, set VRESET ≥ 300 mV below the system empty voltage. Disabling the reset comparator in hibernate (VRESET.Dis=1) saves ~0.5 µA.
- **Temperature compensation:** there is no internal temp sensor — the **host must measure cell temperature** (external sensor) and update `CONFIG.RCOMP` at least once per minute for best accuracy. Budget a temperature source if accuracy over temperature matters.
- **`QSTRT`** should be tied to GND unless a deliberate hardware quick-start is required (quick-start can introduce SOC error and is discouraged for most designs).

## 4. Component Placement, Routing & Layout Best Practices

> The MAX17048 datasheet provides **no dedicated layout-guidelines section** (a consequence of its single-passive, low-speed design). The guidance below combines the datasheet's explicit electrical requirements with standard practice for a precision voltage-sense ADC on an I²C bus. Treat layout items not stated in the datasheet as engineering best-practice (medium confidence).

- **Bypass capacitor placement:** Put the **0.1 µF `VDD`–GND bypass cap directly adjacent to pins 3/4 (`VDD`/`GND`)** with the shortest possible loop. Because SOC is derived from voltage measured at `VDD`, this cap also stabilises the sense node — keep its return tight to the IC `GND`.
- **Voltage-sense (Kelvin) routing:** The device measures the cell to **±7.5 mV/cell**. Route `VDD`/`CELL` to the battery positive **as a dedicated sense trace** to the cell terminal (or pack protection output), **not** tapped off a high-current charge/discharge node where IR drop would corrupt the reading. Likewise reference `GND` to the cell negative terminal as close to the cell as practical. Any series resistance carrying load current between the cell and the sense point appears directly as a voltage (and therefore SOC) error.
- **Exposed pad / ground:** On the TDFN, solder the **exposed pad to GND** and connect it to the ground plane (it is the device ground and heat path). Pin 4 `GND` must still be grounded — do not rely on the pad alone. `CTG` (pin 1) must also be tied to ground.
- **I²C routing:** `SDA`/`SCL` run ≤ 400 kHz — not critical, but keep them away from switching/charge nodes and respect the **400 pF max bus capacitance** (governs pull-up value vs. rise-time). Place pull-ups at the bus master / appropriate rail. The internal `I_PD` pulldowns (~0.2 µA) are for disconnect sensing and do not affect pull-up sizing.
- **`ALRT` line:** open-drain; route to the host interrupt with a pull-up to the host's logic rail (5.5 V tolerant input allows a higher pull-up rail than V_DD).
- **`QSTRT`:** tie directly to GND with a short trace when unused to avoid spurious quick-starts from noise pickup.
- **Pack vs. host side:** the IC can sit on either side. On the **pack side** it sees the cell directly; on the **host side**, account for any series protection FET / fuse resistance between the cell and the sense node when interpreting SOC accuracy.

## 5. Design-Validation Checklist (MAX17048)

- [ ] `VDD` operating voltage stays within **2.5–4.5 V** across all cell states; `CELL` never exceeds 12 V abs-max; logic pins never exceed 6 V (5.5 V recommended).
- [ ] `VDD` **and** `CELL` tied to battery positive (single-cell); `GND` and `CTG` to battery negative / ground.
- [ ] Single **0.1 µF** bypass cap placed directly at `VDD`/`GND` with a short return.
- [ ] Voltage sense (`VDD`/`GND`) Kelvin-routed to the cell terminals — not off a high-current path that would add IR-drop SOC error.
- [ ] `QSTRT` tied to GND (unless hardware quick-start deliberately required).
- [ ] `SDA`, `SCL` pull-ups present (bus idles **high**; total bus C ≤ 400 pF; ≤ 400 kHz); single device per bus (fixed addr `0x6C`/`0x6D`).
- [ ] `ALRT` pulled up and routed to a host interrupt **if** low-SOC / OV / UV / reset alerts are used; otherwise may be left unconnected.
- [ ] TDFN exposed pad soldered to GND plane; pin 4 `GND` and pin 1 `CTG` also grounded.
- [ ] Host firmware plan in place for: model load + `RI` clear at power-up, periodic `RCOMP` temperature compensation (external temp source), and VRESET setting matched to captive vs. removable battery.
- [ ] Power-mode budget validated: active 23 µA / hibernate 3 µA (4 µA max) / sleep < 1 µA; hibernate preferred over sleep if continuous gauging needed.

# BQ24075 — Application Summary

**Manufacturer:** Texas Instruments
**Type:** Standalone 1-Cell Li-Ion Linear Battery Charger with Dynamic Power-Path Management (DPPM)
**Package:** VQFN-16 (RGT), 3.0 mm × 3.0 mm, with exposed thermal pad
**Datasheet:** SLUS810N, Sep 2008 – Rev. Oct 2021 (`docs/components/bq24075/bq24075.md`)

> This summary is a reference for validating a board design that uses the **BQ24075**. It covers electrical, application-circuit, and layout considerations only. Packaging, mechanical, soldering, and marking information is intentionally omitted. Where the datasheet describes the whole BQ2407x family, values below are filtered to the **BQ24075 variant** (5.5 V system output, 6.6 V OVP, 4.3 V DPPM, SYSOFF battery-disconnect option).

---

## 1. Short Description

The BQ24075 is an integrated single-cell **Li-Ion linear charger + system power-path manager** for space-constrained portable products. It runs from a **USB port or AC adapter** and charges at up to **1.5 A**. Its defining feature is **Dynamic Power-Path Management (DPPM)**: the system load on `OUT` is powered *directly and independently* from the input while the battery charges in parallel. This:

- lets the system run even with a **defective, deeply discharged, or absent battery**;
- gives **instant system turn-on** when plugged in (regulated `OUT` even at flat battery);
- reduces battery charge/discharge cycling and enables clean charge termination;
- allows the **battery to supplement** the system when the adapter cannot supply peak load.

BQ24075-specific traits:

- **System output `OUT` regulated to a fixed 5.5 V** (vs. battery-tracking variants like BQ24072).
- **`SYSOFF` input** disconnects the battery from `OUT` (FET Q2 off) — useful for shipping/factory mode, host-side impedance-track fuel gauging (e.g. open-circuit-voltage read), or storage.
- **Fixed internal 10 % charge-termination threshold** (no external ITERM pin; that is the BQ24074 option).
- USB-IF compliant input current limiting (100 mA / 500 mA / resistor-programmable), **VIN-DPM** protection of weak USB sources, **28 V-tolerant** input with **6.6 V OVP**, reverse-current / short-circuit / thermal protection, and an NTC (`TS`) battery-temperature input.

**Typical applications:** TWS charging cases & headphones, gaming accessories, video doorbells / IP cameras, asset trackers, portable medical devices.

### 2.7 Thermal Resistance (VQFN-16 RGT, on JEDEC board)

| Metric | Value | Unit |
|---|---|---|
| θ_JA (junction–ambient) | 44.5 | °C/W |
| θ_JC(top) | 54.2 | °C/W |
| θ_JC(bottom) | 3.8 | °C/W |
| θ_JB (junction–board) | 17.2 | °C/W |

**Power dissipation** while charging (use for `T_J` sizing):

```
P = (V_IN − V_OUT) × (I_OUT + I_BAT) + (V_OUT − V_BAT) × I_BAT
T_J = T_A + P × θ_JA   (keep < 125 °C with margin)
```

> Worst-case dissipation occurs early in the cycle at low battery voltage; the datasheet recommends using **3.4 V** as the minimum V_BAT for max-power calculations (the pack reaches ~3.4 V within ~2 min of fast charge). Design so the **thermal-regulation loop is not normally active**; rely on it only for hot ambient / high-input-voltage corner cases.

**Worked component calculations (datasheet §10.2.2.2):**
- `R_ISET = K_ISET / I_CHG = 890 / 0.8 = 1.11 kΩ → 1.13 kΩ (E96)`
- `R_ILIM = K_ILIM / I_IN-MAX = 1550 / 1.3 = 1.19 kΩ → 1.18 kΩ`
- `R_TMR = t_MAXCHG / (10·K_TMR) = (6.25·3600) / (10·48) = 46.8 kΩ → 46.4 kΩ`

### 3.3 Functional Notes Relevant to the Application

- **DPPM priority:** System load on `OUT` always has priority. When (charge + system) current hits the input limit, `OUT` sags to **V_DPPM (4.3 V)** and the charger current is **automatically throttled** to hold the system up. Termination is disabled while in DPPM.
- **Battery supplement:** If load still exceeds the input limit and `OUT` drops below ~(V_BAT − 40 mV), the battery FET turns fully on to supplement; recovers above ~(V_BAT − 20 mV).
- **VIN-DPM:** In USB100/USB500 modes, if `IN` falls to ~4.5 V the input current is cut back to keep the USB source from collapsing.
- **No input present:** `OUT` is powered from the battery (`BAT`→`OUT` via Q2, ~50 mV dropout at 1 A), unless `SYSOFF` is high.
- **SYSOFF behavior (BQ24075):** high disconnects battery from `OUT`; with an adapter present charging is also disabled and `CHG` stays low. Use for ship-mode / OCV fuel-gauge reads.
- **Half-wave adapters:** charger stays on ≥20 ms (typ) after input drops into sleep, supporting 50 Hz half-rectified adapters — but `IN` must not dip below UVLO; add input capacitance if needed.
- **USB inrush:** keep input capacitance **< 10 µF** to stay within the USB-IF 50 µC inrush limit (the input limit governs current beyond the cap).
- **`ISET` / `ILIM` must never float** (disables charging); **`CE`/`EN1`/`EN2`/`SYSOFF` must never float** (undefined operation).

## 4. Component Placement, Routing & Layout Best Practices

From the datasheet Layout Guidelines (§12.1) and Thermal Considerations (§12.3), plus the TI layout example (Fig. 12-1):

- **Decoupling close to the IC:** Place the **`IN`→GND decoupling cap** and the **`OUT`→GND filter cap(s)** as close as possible to the device, with short runs to `IN`, `OUT`, and the thermal-pad GND. Likewise keep the `BAT` cap close to pins 2/3.
- **Separate signal and power grounds:** Keep low-current GND returns (program resistors, `TS`, status pulls) **separate from the high-current charge/discharge paths**. Use a **single-point (star) ground** that joins small-signal ground and power ground at the thermal pad.
- **Size high-current copper:** The `IN` input path and the `OUT`/`BAT` paths carry up to ~1.5 A — make these traces/pours **wide and short** to avoid IR drops (which directly eat into `OUT` regulation headroom and add heat).
- **Thermal pad = ground + heat sink:** The exposed pad is the **main ground** and primary thermal path. Solder it down and connect it to the system ground plane. Add a **via array** (the TI layout shows a 5–6-via pattern under the pad) into internal/bottom copper to spread heat — needed to actually reach 1.5 A charging without entering thermal regulation. **Do not** use the pad as the device's only ground; pin 8 `VSS` must also be grounded.
- **Program resistors local to their pins:** Place `R_ISET`, `R_ILIM`, `R_TMR` (and the `TS` network) immediately at pins 16/12/14/1 with short returns to VSS — these set charge current, input limit, and timers and should not pick up switching/load noise.
- **Status/EN routing:** Keep `CHG`, `PGOOD`, `EN1`, `EN2`, `CE`, `SYSOFF` traces away from the high-current `OUT`/`BAT` nodes.
- **Input capacitor voltage rating:** For installations exposed to wrong/over-voltage adapters, rate the `IN` cap for the transient (a 16 V ceramic may survive a 30 V transient since ceramics are tested to ~2× rated — verify with the cap maker). Keep total input C < 10 µF for USB inrush compliance.
- Full QFN/SON pad/stencil details are in TI's *QFN/SON PCB Attachment* application note (mechanical — out of scope here).

## 5. Design-Validation Checklist (BQ24075)

- [ ] `IN` stays within **4.35–6.4 V** in normal operation; never exceeds 28 V absolute max; OVP at 6.6 V handled gracefully.
- [ ] `IN`→`OUT` headroom ≥ dropout (≈0.5 V worst case at 1 A) so `OUT` holds 5.5 V.
- [ ] `R_ISET` (590 Ω–8.9 kΩ, **1 % tol.**) sets desired `I_CHG = 890/R_ISET`; pin not floating.
- [ ] `R_ILIM` (1.1 k–8 kΩ) sets input limit; pin not floating (float disables charging).
- [ ] `R_TMR` chosen for required safety-timer duration (or float for default / VSS to disable).
- [ ] `CE`, `EN1`, `EN2`, `SYSOFF` each driven to a defined level — **none floating**; `SYSOFF` low for normal operation.
- [ ] `TS`: 10 kΩ 103AT-2 NTC in pack, **or** fixed 10 kΩ to VSS if unused.
- [ ] `IN` cap 1–10 µF (total input C **< 10 µF** for USB-IF inrush); `OUT` and `BAT` caps 4.7–47 µF ceramic.
- [ ] `PGOOD`/`CHG` open-drains have 1 k–100 kΩ pulls (LED or to host rail).
- [ ] Thermal: `T_J = T_A + P·θ_JA < 125 °C` with margin at worst-case (V_BAT≈3.4 V); thermal pad soldered with via array; not normally in thermal regulation.
- [ ] High-current `IN`/`OUT`/`BAT` copper sized for ≥1.5 A; star ground at thermal pad; decoupling caps adjacent to pins.
